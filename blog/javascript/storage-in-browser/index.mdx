有时候我们需要在浏览器中存储一些数据，比如用户的登录信息，这些数据在下次打开浏览器的时候还需要使用，最常用的情况下，我回去使用 Local/Session Storage，及  Cookie 来存储数据。

打开 Chrome 的开发者工具，可以看到 Application 选项卡，我们在这里可以看到 Local Storage 和 Session Storage，以及 Cookie。 但是会看到其他的一些类似数据库的东西，比如 IndexedDB，Web SQL Database，这些是什么呢？

## IndexedDB

简单来说，IndexedDB就是一种浏览器中的数据库，可以在本地存储大量数据，提供了类似于SQL数据库的许多功能，如创建、读取、更新和删除数据等。它是基于异步事件驱动的编程模型，因此可以更好地处理大量数据和复杂的查询操作。

如果需要完成 *网站里先访问*， *缓存* 或者 *应用程序数据存储* 等功能，那么 IndexedDB 就是一个不错的选择。它支持事务处理和复杂的索引查询，可以存储各种类型的数据，包括二进制数据、对象、数组等等。
